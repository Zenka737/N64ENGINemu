# N64ENGINemu

Эмулятор Nintendo 64, разрабатываемый с нуля на C/C++.

## О проекте

N64ENGINemu нацелен на точную и производительную эмуляцию консоли Nintendo 64:
эмуляцию центрального процессора VR4300 (MIPS R4300i), сопроцессора RSP,
графического процессора RDP, аудио- и вход-подсистем, а также загрузку и
исполнение образов ROM в форматах `.z64` / `.n64` / `.v64`.

## Статус

Проект находится на самой ранней стадии — идёт планирование архитектуры и
базовой структуры кода. Готового к использованию эмулятора пока нет.

## Архитектура (план)

- **CPU** — интерпретатор/динамическая рекомпиляция VR4300 (MIPS R4300i, 64-бит), TLB, кэши, обработка исключений.
- **RSP** — эмуляция Reality Signal Processor и микрокода (audio/graphics ucode).
- **RDP** — растеризация, текстурирование, цветовой комбайнер, кадровый буфер; вывод через OpenGL/Vulkan бэкенд.
- **Память и шина** — RDRAM, регистры RCP (MI, SP, DP, VI, AI, PI, RI, SI), картридж и PIF (загрузчик/аутентификация).
- **Аудио** — HLE аудио-микрокодов и вывод звука.
- **Ввод** — эмуляция контроллера N64 и периферии (Rumble Pak, Controller Pak).
- **Сохранения** — SRAM/EEPROM/FlashRAM.

## Разработка через агентов Claude Code

В `.claude/agents/` описаны специализированные агенты для этого репозитория:

- **planner** (модель `fable`) — планировщик: декомпозиция задач, роадмап, приоритизация подсистем.
- **cpu-core** (модель `sonnet`) — ядро эмуляции: CPU, RSP, память, шина.
- **graphics-rdp** (модель `sonnet`) — графика: эмуляция RDP и видео-бэкенд.

## Сборка

Требуется CMake >= 3.16 и компилятор с поддержкой C++20.

```sh
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
ctest --test-dir build --output-on-failure
```

Запуск: `./build/n64enginemu <path-to-rom>`.

## CI

На каждый Pull Request автоматически запускаются (см. `.github/workflows/ci.yml`):

- сборка проекта под GCC и Clang с флагами `-Wall -Wextra -Wpedantic -Werror`;
- юнит-тесты через CTest;
- проверка форматирования (`clang-format --dry-run --Werror`);
- статический анализ (`clang-tidy`, warnings-as-errors).

PR не должен мержиться, пока все эти проверки не пройдены.
